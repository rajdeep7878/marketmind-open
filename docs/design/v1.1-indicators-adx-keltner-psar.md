# v1.1 indicators: ADX + Keltner Channels + PSAR

A combined design doc for three indicators added together (2026-05-23)
as a focused sweep after Supertrend (2026-05-22) proved the per-indicator
pattern. All three reuse the `ta` library — no hand-roll needed.

## Context

`v1.1-todos.md` "Indicator whitelist expansion" lists ADX, Parabolic SAR,
and Keltner Channels as the remaining post-Supertrend additions. The
Supertrend addition surfaced one tooling insight (check `ta` first — most
common indicators are already in it) and one operational pattern (per-
indicator narrow `_detect_axes` widening, not a generic refactor). This
doc captures the Q1/Q2/Q3 decisions for the three indicators in one pass.

## ADX (Average Directional Index)

### Q1 — Output shape: single-output scalar

The ADX value (0–100; > 25 is the conventional "trending" threshold).
The `ta` library also exposes `adx_pos()` (+DI) and `adx_neg()` (-DI) as
separate series — useful for direction-of-trend reads — but the dominant
use case is "ADX > N = trending" which only needs the ADX line.
Multi-output (adx/+DI/-DI) is a future addition if a strategy needs the
directional indicators; single-output is the lower-friction starting
point.

### Q2 — Implementation: `ta.trend.ADXIndicator`

`ta.ADXIndicator(high, low, close, window=period).adx()` — returns a
pandas Series, no hand-rolling. Reference test: compare against a direct
`ta.ADXIndicator` call on a synthetic fixture (the `ta` library is the
reference because our impl wraps it; the test asserts the wrapping is
correct).

### Q3 — Bounds: `period` only

`period [2, 100]` — mirrors ATR / RSI (Wilder-smoothed indicator family).
Default: 14 (the canonical Wilder default).

## Keltner Channels

### Q1 — Output shape: multi-output (upper/middle/lower)

Mirrors `bollinger` exactly — returns a `pd.DataFrame` with the three
columns; specs select via `IndicatorExpr.component`. The composition is
identical to Bollinger conceptually (middle = EMA(window); upper/lower =
middle ± multiplier × ATR(window_atr)).

### Q2 — Implementation: `ta.volatility.KeltnerChannel`

`ta.KeltnerChannel(high, low, close, window=period, window_atr=atr_period,
multiplier=multiplier, original_version=False)`. The
`original_version=False` flag selects the modern EMA-based middle band
(Linda Raschke 1980s variant) over `ta`'s default 1960 SMA-based original
— modern Keltner is what every TradingView / Stockcharts user expects
and the 1960 SMA variant is essentially never used in practice. The
three components come from `.keltner_channel_hband()` / `_mband()` /
`_lband()`, wrapped into a DataFrame matching the `bollinger` pattern.

### Q3 — Bounds: three numeric params

| Param | Bound | Reasoning |
|---|---|---|
| `period` | `[5, 200]` | Mirrors `bollinger.period` (also an EMA window). |
| `atr_period` | `[2, 100]` | Mirrors `ATR`'s bound. |
| `multiplier` | `[1.0, 10.0]` | Mirrors Supertrend's bound; classic Keltner is 2.0. |

**Defaults:** `(period=20, atr_period=10, multiplier=2.0)` — matches
`ta`'s defaults and the textbook Keltner.

## PSAR (Parabolic SAR)

### Q1 — Output shape: multi-output (value + direction)

Mirrors Supertrend's two-output shape. `value` = the SAR price (always
non-NaN once warm); `direction` = `+1.0` when SAR is below price
(uptrend), `-1.0` when above (downtrend) — derived from `ta`'s
`psar_up_indicator()` / `psar_down_indicator()` masks. A PSAR strategy
typically reads `psar.direction` as the trend gate, or `psar.value` as a
trailing stop level.

### Q2 — Implementation: `ta.trend.PSARIndicator`

`ta.PSARIndicator(high, low, close, step, max_step)`. Use `psar()` for
`value`; derive `direction` from `psar_up_indicator()` (`+1.0` where
True, `-1.0` elsewhere among the non-NaN bars). PSAR is stateful at the
indicator level — the acceleration factor accumulates while a trend
persists — but the recursion lives inside `ta`'s implementation, exactly
like Supertrend's lives inside our hand-rolled function. No special
`build_signals` threading; the translator dispatches PSAR like any other
multi-output indicator.

### Q3 — Bounds: two numeric params

| Param | Bound | Reasoning |
|---|---|---|
| `step` | `[0.01, 0.1]` | Acceleration factor; canonical 0.02 (Wilder). |
| `max_step` | `[0.1, 1.0]` | Acceleration cap; canonical 0.2 (Wilder). |

**Defaults:** `(step=0.02, max_step=0.2)` — Wilder's original.

## Parameter sweep — narrow per-indicator widening

Following the Supertrend pattern: `_detect_axes` currently detects
`period` on the hardcoded set `{sma, ema, rsi, wma}`. The per-indicator
add for this batch: extend the set to include `adx` and `keltner` (both
with `period` as the swept axis). PSAR's `step` / `max_step` are not
period-like and remain un-swept, consistent with Supertrend's
`multiplier` and Bollinger's `std_dev`. Keltner's `atr_period` is also
not swept by this narrow add — the same gap as Supertrend's `atr_period`.

**Logged as a v1.1 follow-up:** a generic `_detect_axes` widening that
iterates `INDICATOR_RULES[*].numeric` and detects every period-like
param across every indicator (including `atr_period`). It would re-run
on previously-seeded strategies' next overfitting call and surface MORE
axes than they have today — that's not a live-trader regression
(overfitting is offline) but it is a meaningful tool-behaviour change
worth its own deliberate scoping. Out of scope for this batch.

## Order of implementation

ADX first (single-output scalar — simplest; validates the batched
per-indicator pattern in this session). Then Keltner (multi-output,
mirrors bollinger exactly — validates the new-indicator multi-output
path). Then PSAR (stateful, value + direction, mirrors Supertrend —
most complex, benefits from the prior two having proven the `ta`-reuse
path).

## Status

Phase 1 complete — all three designs settled, no design fork required
user input. The `ta` library has clean interfaces for all three; no
indicator needs hand-rolling. Implementation follows in per-indicator
commit sets (enum + bounds, impl, translator, prompt — 4 each, 12 total).
