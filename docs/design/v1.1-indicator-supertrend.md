# v1.1 indicator: Supertrend

## Context

`v1.1-todos.md` "Indicator whitelist expansion" logs Supertrend as the
priority indicator: on 2026-05-21 a Boring Edge Supertrend article
extracted as a Tier-1 EMA-crossover fallback because Supertrend was not
on the 15-indicator whitelist. Supertrend is stateful at the indicator
level — its trend direction at bar *t* depends on the direction at *t-1* —
so it composes naturally with v2's `regime_state` primitive.

This doc records the three design decisions resolved during Phase 1
discovery, before implementation.

## Q1 — Output shape: two components (`value` + `direction`)

**Decision: Supertrend is a multi-output indicator returning a DataFrame
with two columns — `value` and `direction` — selected via
`IndicatorExpr.component`.**

The codebase already has this pattern. `bollinger` returns
`upper/middle/lower`, `macd` returns `line/signal/hist`, `stochastic`
returns `k/d`; each is an `INDICATOR_RULES` entry with a `components`
tuple, and the translator's `_select_component` picks the requested
column. Supertrend fits it exactly:

- `value` — the active Supertrend line (the trailing band price). Usable
  as a stop level or a threshold in a `compare`.
- `direction` — `+1.0` when bullish (line below price), `-1.0` when
  bearish. A strategy reads `supertrend.direction` to drive a regime:
  `compare(supertrend.direction > 0)` is the bullish-trend gate, and it
  slots into `regime_state` for trend-following entries.

No new interface is needed — the two-output requirement from the
v1.1-todos note is satisfied by the existing multi-output mechanism.

## Q2 — Implementation: hand-rolled (no library ships it)

**Decision: hand-implement Supertrend in `backtest/indicators.py`.**

Discovery checked every indicator library in the environment:
`vectorbt` 0.28.5, `ta`, `pandas_ta`, `talib`. **None ships a Supertrend
indicator** — vectorbt's `STX`/`OHLCSTX` are stop-exit signal tools, not
Supertrend; `ta.trend` has none; `pandas_ta` and `talib` are not
installed. So the brief's "reuse vectorbt's Supertrend" branch does not
apply — the "else: hand-implement" branch does.

Hand-rolling is also the consistent choice: `indicators.py` already
hand-rolls VWAP and the candle patterns, and it reuses `ind.atr` (the
`ta`-backed Wilder ATR) for the ATR term.

**Reference test:** with no library Supertrend to diff against, the test
uses (a) a hand-computed known-value fixture and (b) an independent
naive reference implementation written in the test file — a slow,
obviously-correct bar-by-bar loop — compared against the production
implementation. The Supertrend formula is canonical and deterministic,
so a hand-verified reference is rigorous.

## Algorithm

    hl2          = (high + low) / 2
    atr          = ATR(atr_period)               # Wilder's, via ind.atr
    basic_upper  = hl2 + multiplier * atr
    basic_lower  = hl2 - multiplier * atr

    final_upper[t] = basic_upper[t]  if basic_upper[t] < final_upper[t-1]
                                        or close[t-1] > final_upper[t-1]
                     else final_upper[t-1]
    final_lower[t] = basic_lower[t]  if basic_lower[t] > final_lower[t-1]
                                        or close[t-1] < final_lower[t-1]
                     else final_lower[t-1]

    # direction & line — recursive flip
    if direction[t-1] == -1:          # was bearish, line tracked final_upper
        bullish = close[t] > final_upper[t]
    else:                             # was bullish
        bullish = close[t] >= final_lower[t]
    direction[t] = +1 if bullish else -1
    value[t]     = final_lower[t] if bullish else final_upper[t]

The recursion is sequential and lives entirely inside the indicator
function — exactly like `ema` (also recursive). The translator dispatches
Supertrend like `bollinger` (`ind.supertrend(df, ...)` → DataFrame →
`_select_component`); no state threads through `build_signals` /
`build_signals_stateful`. Warmup bars (ATR undefined) emit `NaN` for both
columns; the first valid bar seeds `direction` from `close` vs `hl2`.

## Q3 — Parameter bounds

Two parameters: `atr_period` (int) and `multiplier` (float).

**Validation bounds** (`INDICATOR_RULES` — the hard reject-garbage limits):

| Param | Bound | Reasoning |
|---|---|---|
| `atr_period` | `[2, 100]` | Mirrors the `ATR` indicator's own bound — Supertrend's `atr_period` *is* an ATR period; a tighter cap would reject Supertrend configs that bare `ATR` accepts. The schema philosophy is generous bounds. |
| `multiplier` | `[1.0, 10.0]` | Generous. Below ~1.0 the bands hug price and the line whipsaws every bar; above ~10 they are so wide the trend never flips. 1.0–10.0 brackets every defensible value. |

**Defaults:** `atr_period=10`, `multiplier=3.0` — the canonical Supertrend
default (the original and most-cited configuration).

**Literature-typical range** (sweep guidance, not a hard bound):
`atr_period` 7–14, `multiplier` 1.5–4.0. Parameter sweeps explore around
the extracted value within the validation bound; the generous validation
bound and the narrower typical sweep range are deliberately distinct.

## Phase 3 — validation against the original gap

A Boring Edge-style Bitcoin Supertrend 4H strategy was re-extracted after
deploying commits 2–5 (worker rebuilt). Result:

- Verdict `fully_extractable` — the original had been
  `partially_extractable`, flagging Supertrend as missing and falling
  back to an EMA(10) crossover.
- The spec uses the `supertrend` indicator: `crossover` of the
  `direction` component across 0 (atr_period 10, multiplier 3). No EMA
  fallback. **The whitelist gap is closed.**
- The spec validates and backtests cleanly: 160 trades over 2020–2026,
  +599% total / Sharpe 0.95 / profit factor 1.31 (alpha −3.8% vs B&H —
  a trend-follower trailing a historic bull market, as expected).

**The spec is Tier-1, not Tier-2.** A pure Supertrend strategy is
stateless *at the spec level* — `crossover(supertrend.direction)` —
because Supertrend carries its trend state inside the indicator's own
recursion. The brief anticipated Tier-2 "since Supertrend composes with
regime_state", but wrapping a self-latching indicator in `regime_state`
is the degenerate pattern the prompt explicitly warns against (it
reduces to exactly this crossover). Tier-1 is the correct, faithful
extraction. Supertrend *can* still feed a genuinely hysteretic
`regime_state` as one input — that is just not what a plain Supertrend
strategy needs.

## Known limitation — parameter sweep

`_detect_axes` (the overfitting parameter sweep) detects `period` on
sma/ema/rsi/wma only; Supertrend's `atr_period`/`multiplier` are not
auto-swept — consistent with atr/bollinger/macd/highest/lowest also
being un-swept. Widening the sweep to volatility-band indicators is a
separate follow-up, not part of this whitelist addition.
