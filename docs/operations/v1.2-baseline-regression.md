## Ad-hoc perf baselines (iterative engine, Modern Turtle, 3-run median)

Not a committed test — measured directly via `run_iterative_backtest()`
on the three fixtures, three warm runs each. The committed perf-
regression tests above measure the same code path but with a single
timed run; this table records the median/max for v1.2 sub-phases to
compare against.

| timeframe | bars | trades | min | **median** | max |
|---|---|---|---|---|---|
| 4H | 13,985 | 160 | 0.223 s | **0.314 s** | 0.362 s |
| 1H | 55,912 | 665 | 1.020 s | **1.044 s** | 1.061 s |
| 15m | 223,527 | 3,198 | 4.010 s | **4.246 s** | 4.324 s |

**Linear-scaling check** (medians, relative to 4H):

- 1H / 4H = 1.044 / 0.314 = **3.32×** vs linear-pred 4× (slightly
  better than linear — small-N edge effects).
- 15m / 4H = 4.246 / 0.314 = **13.52×** vs linear-pred 16× (within the
  same envelope; the per-bar work is bounded by the indicator + ratchet
  loop, not the bar count).

Confirms the Phase B.4 / B.8 finding: the iterative engine's per-bar
work is O(indicators × bar), not O(bar²). Any v1.2 sub-phase that
materially regresses these numbers (especially 15m climbing toward
8 s) is a red flag the sign-off check should catch.

## Sign-off acceptance reference (from §7 of the design doc)

For v1.2.F to close, the final state must satisfy all 8 criteria
against this baseline:

1. **Test count:** ≥ 1130 passes (1099 baseline + ~31 new tests
   estimate). Allow ±5 slack; document any test removal or merge.
2. **Drift parity:** all 9 gates green (5 pre-existing + 5 new) with
   zero divergence reported.
3. **v1 regression:** the 3 production strategies' spec_json from
   `trader_strategy_versions` round-trips through
   `StrategySpec.model_validate` + re-serialise with no diff. Trader
   cycle pattern unchanged.
4. **Performance:** B.4 1H Turtle < 5 s threshold; B.8 15m Turtle
   < 8 s threshold; full suite within ±20 % of 84 s baseline (target
   < 100 s).
5. **Ruff + pyright:** clean on `api/src`, `workers/src`, `shared/src`.
6. **Schema artifacts:** generated
   `web/src/types/generated/schemas.json` regenerated and committed;
   TS bindings match the new Pydantic shapes.
7. **Container rebuild documented:** `docs/operations/v1.2-complete.md`
   explicitly names the three-container rebuild step + verification
   commands.
8. **Bot regression:** the running paper bot (3 strategies, Modern
   Turtle warmup) unchanged through the entire v1.2 work — none of the
   v1.2 sub-phases require a trader_worker restart until the final
   post-merge rebuild.

This baseline is the comparison point for criteria 1, 2, 4, 5, and 8.
Criterion 3 is measured separately (round-trip test of the 3
production spec_json blobs); criterion 6 is the artifact-existence
check; criterion 7 is the doc-presence check.
